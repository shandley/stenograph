     1→# CodonGhost: Nucleotide-Informed Synthetic MSA Generation for Viral Protein Structure Prediction
     2→
     3→## Abstract
     4→
     5→Structure prediction for viral proteins is frequently limited by sparse multiple sequence alignments (MSAs), a consequence of rapid viral evolution and the underrepresentation of viral diversity in sequence databases. We propose CodonGhost, a hybrid approach that generates synthetic MSAs by simulating sequence evolution in codon space using well-parameterized nucleotide evolutionary models, while filtering and weighting synthetic sequences using protein-level fitness landscapes learned from protein language models. This approach leverages decades of empirical work on viral molecular evolution to generate biologically plausible sequence diversity, grounded in the specific evolutionary dynamics of the viral family under study.
     6→
     7→## Background and Motivation
     8→
     9→### The MSA Depth Problem
    10→
    11→AlphaFold2 and related structure prediction methods extract coevolutionary signal from MSAs to infer residue-residue contacts. Performance degrades substantially when:
    12→- Effective sequence count (Neff) is low
    13→- The protein family is taxonomically restricted
    14→- The sequence is an "orphan" with few detectable homologs
    15→
    16→Viral proteins are disproportionately affected because:
    17→1. Rapid evolution erodes detectable homology
    18→2. Many viral lineages are poorly sampled
    19→3. Novel viruses from metagenomic surveys often lack close relatives
    20→4. Some viral proteins have no cellular homologs (truly novel folds)
    21→
    22→### Current Approaches and Their Limitations
    23→
    24→**Protein language model approaches** (ESMFold, OmegaFold):
    25→- Bypass MSA requirement entirely
    26→- Work well for some cases but sacrifice coevolutionary information
    27→- Lower accuracy than MSA-based methods when good MSAs exist
    28→
    29→**Synthetic MSA generation** (MSAGPT, MSA-Generator, GhostFold):
    30→- Generate virtual homologs using protein language models
    31→- Work entirely in amino acid space
    32→- Learn generic protein statistics, not virus-specific evolution
    33→- Cannot incorporate known evolutionary parameters
    34→
    35→**Nucleotide-level evolutionary simulation**:
    36→- Well-developed theory and parameterization
    37→- Extensively validated for viral systems
    38→- Not previously applied to MSA augmentation for structure prediction
    39→
    40→### The Opportunity
    41→
    42→Viral molecular evolution is arguably the best-parameterized evolutionary system:
    43→- Mutation rates measured empirically for many viruses
    44→- Transition/transversion biases well characterized
    45→- Codon usage bias documented across host systems
    46→- dN/dS variation extensively studied
    47→- Recombination rates known for many families
    48→
    49→This knowledge is currently unused in structure prediction pipelines.
    50→
    51→## Proposed Approach: CodonGhost
    52→
    53→### Core Concept
    54→
    55→Generate synthetic sequence diversity by:
    56→1. **Mutating in codon/nucleotide space** using parameterized evolutionary models
    57→2. **Selecting in protein space** using fitness landscapes from protein language models
    58→3. **Combining signals** to produce synthetic MSAs that are both evolutionarily realistic and structurally informative
    59→
    60→### Pipeline Architecture
    61→
    62→```
    63→┌─────────────────────────────────────────────────────────────────┐
    64→│                         INPUT LAYER                              │
    65→├─────────────────────────────────────────────────────────────────┤
    66→│  Sparse viral MSA (coding sequences)                            │
    67→│  Phylogenetic tree estimate (or generate de novo)               │
    68→│  [Optional] Known evolutionary parameters for viral family      │
    69→└─────────────────────────────────────────────────────────────────┘
    70→                              │
    71→                              ▼
    72→┌─────────────────────────────────────────────────────────────────┐
    73→│                    PARAMETERIZATION LAYER                        │
    74→├─────────────────────────────────────────────────────────────────┤
    75→│  1. Estimate/refine nucleotide substitution model               │
    76→│     - GTR parameters from alignment                             │
    77→│     - Site-rate heterogeneity (Γ distribution)                  │
    78→│     - Codon model parameters (GY94/MG94)                        │
    79→│                                                                 │
    80→│  2. Estimate tree topology and branch lengths                   │
    81→│     - Or use provided tree                                      │
    82→│     - Optionally extend tree with synthetic ancestors/lineages  │
    83→│                                                                 │
    84→│  3. Load/initialize fitness oracles                             │
    85→│     - cdsFM for codon-level fitness                             │
    86→│     - ESM-2 for protein-level fitness                           │
    87→│     - [Optional] Family-specific Potts model if Neff sufficient │
    88→└─────────────────────────────────────────────────────────────────┘
    89→                              │
    90→                              ▼

    92→│                     SIMULATION LAYER                             │

    95→│                                                                 │

    98→│     - Use reconstructed ancestral sequence                      │
    99→│                                                                 │
   100→│  2. Simulate divergence along branch                            │
   101→│     - Propose mutations according to nucleotide/codon model     │
   102→│     - Calculate acceptance probability:                         │
   103→│                                                                 │
   104→│       P(accept) ∝ P_nuc(mutation) ×                            │

   122→│     - Should be straightforward given simulation from alignment │

   128→│  4. [Optional] Reweight sequences                               │
   129→│     - By protein-level fitness score                            │

   144→## Detailed Methods
   145→
   146→### 1. Nucleotide Model Parameterization
   147→
   148→For each input alignment, estimate parameters for a codon substitution model:
   149→
   150→**Base model**: GY94 (Goldman & Yang 1994) or MG94 (Muse & Gaut 1994)
   151→- κ: transition/transversion ratio
   152→- ω: global dN/dS ratio (or site-specific ω for M3/M8 models)
   153→- π: equilibrium codon frequencies (F61, F3x4, or F1x4)
   154→
   155→**Extensions for viral sequences**:
   156→- Site-rate heterogeneity: Γ-distributed rates across sites
   157→- Codon usage bias: host-specific frequency parameters
   158→- Multiple nucleotide substitutions: ECM-style instantaneous doublet/triplet rates
   159→
   160→**Implementation options**:
   161→- PAML (codeml) for parameter estimation
   162→- HyPhy for more complex models
   163→- IQ-TREE for integrated tree + model estimation
   164→- Custom implementation for tight pipeline integration
   165→
   166→### 2. Fitness Landscape Integration
   167→
   168→#### Tier 1: Codon-level fitness (cdsFM)
   169→
   170→Use codon foundation models to score codon choices:
   171→
   172→```python
   173→def codon_fitness(coding_seq, model='cdsFM'):
   174→    """
   175→    Returns pseudo-likelihood score for coding sequence.
   176→    Higher = more fit according to learned codon preferences.
   177→    """
   178→    # Mask each codon, predict, score
   179→    log_likelihood = 0
   180→    for i in range(0, len(coding_seq), 3):
   181→        codon = coding_seq[i:i+3]
   182→        context = mask_position(coding_seq, i)
   183→        p_codon = model.predict(context)[codon]
   184→        log_likelihood += log(p_codon)
   185→    return log_likelihood
   186→```
   187→
   188→#### Tier 2: Protein-level fitness (ESM-2)
   189→
   190→Score translated protein for structural plausibility:
   191→
   192→```python
   193→def protein_fitness(protein_seq, model='esm2_t33_650M'):
   194→    """
   195→    Returns pseudo-perplexity from ESM-2.
   196→    Lower perplexity = more structurally plausible.
   197→    """
   198→    # Mask each position, compute probability of true AA
   199→    log_likelihood = 0
   200→    for i, aa in enumerate(protein_seq):
   201→        context = mask_position(protein_seq, i)
   202→        p_aa = model.predict(context)[aa]
   203→        log_likelihood += log(p_aa)
   204→    return log_likelihood  # Or return -perplexity
   205→```
   206→
   207→#### Tier 3: Family-specific epistasis (if sufficient data)
   208→
   209→If Neff > ~100, fit a Potts model to capture pairwise epistasis:
   210→
   211→```python
   212→def epistatic_fitness(protein_seq, potts_model):
   213→    """
   214→    Returns Potts model energy (statistical energy).
   215→    More negative = more consistent with natural sequences.
   216→    """
   217→    energy = sum(potts_model.h[i, aa] for i, aa in enumerate(protein_seq))
   218→    energy += sum(potts_model.J[i, j, aa_i, aa_j] 
   219→                  for (i, aa_i), (j, aa_j) in pairs(protein_seq))
   220→    return -energy  # Convert to fitness (higher = better)
   221→```
   222→
   223→### 3. Simulation Algorithm
   224→
   225→#### Metropolis-Hastings in Codon Space
   226→
   227→```python
   228→def simulate_divergent_sequence(
   229→    starting_seq,      # Coding sequence (nucleotides)
   230→    target_divergence, # Desired protein-level % identity
   231→    nuc_model,         # Parameterized nucleotide/codon model
   232→    fitness_fn,        # Combined fitness function
   233→    temperature=1.0    # Selection stringency
   234→):
   235→    current_seq = starting_seq
   236→    current_fitness = fitness_fn(current_seq)
   237→    
   238→    while protein_identity(current_seq, starting_seq) > target_divergence:
   239→        # Propose mutation according to nucleotide model
   240→        site = sample_site(nuc_model.site_rates)
   241→        old_codon = current_seq[site:site+3]
   242→        new_codon = sample_mutation(old_codon, nuc_model)
   243→        
   244→        # Skip if introduces stop codon
   245→        if translates_to_stop(new_codon):
   246→            continue
   247→        
   248→        # Calculate acceptance probability
   249→        proposed_seq = substitute(current_seq, site, new_codon)
   250→        proposed_fitness = fitness_fn(proposed_seq)
   251→        
   252→        # Fitness ratio (with temperature)
   253→        delta_fitness = (proposed_fitness - current_fitness) / temperature
   254→        
   255→        # Nucleotide model probability ratio
   256→        nuc_ratio = nuc_model.rate(old_codon, new_codon) / \
   257→                    nuc_model.rate(new_codon, old_codon)
   258→        
   259→        # Metropolis-Hastings acceptance
   260→        accept_prob = min(1, nuc_ratio * exp(delta_fitness))
   261→        
   262→        if random() < accept_prob:
   263→            current_seq = proposed_seq
   264→            current_fitness = proposed_fitness
   265→    
   266→    return current_seq
   267→```

   269→#### Combined Fitness Function
   270→
   271→```python
   272→def combined_fitness(coding_seq, weights={'codon': 0.3, 'protein': 0.7}):
   273→    """
   274→    Combine multiple fitness signals.
   275→    Weights can be tuned or learned.
   276→    """
   277→    protein_seq = translate(coding_seq)
   278→    
   279→    score = 0
   280→    score += weights['codon'] * codon_fitness(coding_seq)
   281→    score += weights['protein'] * protein_fitness(protein_seq)
   282→    
   283→    # Optional: add Potts model if available
   284→    if potts_model is not None:
   285→        score += weights.get('epistasis', 0.2) * epistatic_fitness(protein_seq)
   286→    
   287→    return score
   288→```

   290→### 4. Tree Augmentation Strategies
   291→
   292→To generate diverse synthetic sequences, extend the phylogenetic tree:
   293→
   294→**Strategy A: Extend terminal branches**
   295→- For each real sequence, simulate additional divergence
   296→- Creates "near neighbors" of existing sequences
   297→
   298→**Strategy B: Add ancestral reconstructions**
   299→- Reconstruct internal node sequences
   300→- Simulate divergence from ancestors along novel branches
   301→- Creates sequences exploring "evolutionary paths not taken"
   302→
   303→**Strategy C: Deep branch extension**
   304→- Extend branches from root or deep nodes
   305→- Creates maximally divergent plausible sequences
   306→
   307→**Strategy D: Star topology from consensus**
   308→- Generate consensus/ancestor sequence
   309→- Simulate independent lineages at various divergence levels
   310→- Ensures even coverage of sequence space

   312→### 5. MSA Assembly and Weighting
   313→
   314→After generating synthetic sequences:
   315→
   316→```python
   317→def assemble_augmented_msa(
   318→    natural_seqs,
   319→    synthetic_seqs,
   320→    max_synthetic=1000,
   321→    reweight=True
   322→):
   323→    # Score all synthetic sequences
   324→    scored_synthetic = [
   325→        (seq, protein_fitness(translate(seq)))
   326→        for seq in synthetic_seqs
   327→    ]
   328→    
   329→    # Filter to top-scoring synthetic sequences
   330→    scored_synthetic.sort(key=lambda x: x[1], reverse=True)
   331→    top_synthetic = [s[0] for s in scored_synthetic[:max_synthetic]]
   332→    
   333→    # Translate to protein
   334→    natural_proteins = [translate(s) for s in natural_seqs]
   335→    synthetic_proteins = [translate(s) for s in top_synthetic]
   336→    
   337→    # Combine (natural first for implicit weighting)
   338→    combined = natural_proteins + synthetic_proteins
   339→    
   340→    if reweight:
   341→        # Apply sequence reweighting (e.g., Henikoff weights)
   342→        # Or custom weighting favoring natural sequences
   343→        weights = compute_weights(combined, natural_count=len(natural_proteins))
   344→        return combined, weights
   345→    
   346→    return combined, None
   347→```

Purpose: A novel approach to improve protein structure prediction for viral proteins by generating synthetic Multiple Sequence Alignments (MSAs) using nucleotide-level evolutionary simulation.
