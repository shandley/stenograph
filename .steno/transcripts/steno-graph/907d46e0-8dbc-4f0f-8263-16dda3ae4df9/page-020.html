   346→### steno:undo
   347→
   348→Undo the last command or a specific node.
   349→
   350→**Soft undo (default)** — removes from graph, keeps files:
   351→
   352→```
   353→> steno:undo
   354→
   355→Undoing n_005: ch:@data.csv +normalize
   356→  Removed from session graph.
   357→  Files kept: normalized.csv
   358→
   359→  Dependent nodes (will be orphaned):
   360→    n_006: viz:pca ^
   361→
   362→  Use steno:redo to restore.
   363→```
   364→
   365→**Hard undo** — also reverts file changes:
   366→
   367→```
   368→> steno:undo!
   369→
   370→Undoing n_005: ch:@data.csv +normalize
   371→  Removed from session graph.
   372→  Deleted: normalized.csv
   373→  Reverted: data.csv (restored from before edit)
   374→
   375→  Dependent nodes removed: n_006
   376→```
   377→
   378→**Undo specific node:**
   379→
   380→```
   381→> steno:undo n_003
   382→
   383→Undoing n_003: viz:heatmap @samples.csv
   384→  Removed from session graph.
   385→  Files kept: heatmap.png
   386→```
   387→
   388→**Behavior:**
   389→
   390→1. Read `.steno/current-session.json`
   391→2. Find target node (last if none specified)
   392→3. Check for dependent nodes (nodes that reference this one via `^`)
   393→4. Soft undo:
   394→   - Remove node from session
   395→   - Move to `undone` array (for redo)
   396→   - List created files (user can delete manually)
   397→   - Warn about orphaned dependents
   398→5. Hard undo (`!`):
   399→   - Remove node and all dependents
   400→   - Delete created files
   401→   - Revert modified files (if backup exists)
   402→
   403→**File handling:**
   404→
   405→For hard undo, steno needs file backups. When a command modifies a file:
   406→- Before: Save `.steno/backups/{node_id}/{filename}`
   407→- After: Record in node's `backups` field
   408→
   409→```json
   410→{
   411→  "id": "n_005",
   412→  "raw": "ch:@data.csv +normalize",
   413→  "outputs": ["normalized.csv"],
   414→  "modified": ["data.csv"],
   415→  "backups": {
   416→    "data.csv": ".steno/backups/n_005/data.csv"
   417→  }
   418→}
   419→```
   420→
   421→Hard undo restores from backup and deletes outputs.
   422→
   423→### steno:redo
   424→
   425→Restore the last undone node.
   426→
   427→```
   428→> steno:redo
   429→
   430→Restoring n_005: ch:@data.csv +normalize
   431→  Added back to session graph.
   432→
   433→  Note: Files were not deleted, so outputs still exist.
   434→```
   435→
   436→**Behavior:**
   437→
   438→1. Check `undone` array in current session
   439→2. Pop the last undone node
   440→3. Add it back to the session
   441→4. Note: Does not re-execute the command, just restores tracking
   442→

   444→
   445→```
   446→> steno:redo
   447→
   448→Restoring n_005: ch:@data.csv +normalize
   449→  Added back to session graph.
   450→
   451→  ⚠ Files were deleted. Re-run command to recreate:
   452→    ch:@data.csv +normalize

   700→## Branching
   701→
   702→Branches allow exploring alternative approaches without losing work.
   703→
   704→### fork:name
   705→
   706→Create a new branch from the current node.
   707→
   708→```
   709→> fork:caching
   710→
   711→Created branch "caching" from n_002
   712→Switched to branch "caching"
   713→```
   714→
   715→**Behavior:**
   716→1. Read `.steno/graph.json`
   717→2. Create a new branch object:
   718→   ```json
   719→   {
   720→     "name": "caching",
   721→     "parentNode": "n_002",
   722→     "parentBranch": "main",
   723→     "created": "ISO timestamp",
   724→     "nodes": [],
   725→     "status": "active"
   726→   }
   727→   ```
   728→3. Add to `branches` array in graph.json
   729→4. Set `currentBranch` to the new branch name
   730→5. Subsequent nodes will be tagged with this branch
   731→

   733→
   734→Switch to an existing branch.
   735→
   736→```
   737→> switch:main
   738→
   739→Switched to branch "main"
   740→Current node: n_002
   741→```
   742→
   743→**Behavior:**
   744→1. Verify branch exists in graph.json
   745→2. Set `currentBranch` to target branch
   746→3. The `^` reference now resolves to the last node on that branch
   747→

   749→
   750→List all branches with their status.
   751→
   752→```

   763→- Current branch marked with *

   770→Compare two branches side by side.

   806→

   966→
   967→When commands fail or are malformed, respond with clear, actionable messages. Never silently fail or guess user intent.

   972→> [command]
   973→
   974→⚠ [error type]: [what went wrong]

   979→
   980→| Error | Message |
   981→|-------|---------|
   982→| `dx:` (no target) | `⚠ Missing target: dx: requires a file or reference. Try: dx:@file.csv` |
   983→| `ch:` (no target) | `⚠ Missing target: ch: requires something to change. Try: ch:@file.py +feature` |
   984→| `fork:` (no name) | `⚠ Missing branch name: fork: requires a name. Try: fork:experiment` |

   988→
   989→```
   990→> dx:@missing.csv

   998→1. Check if file exists at the path
   999→2. If not, look for similar filenames (fuzzy match)
  1000→3. Suggest alternatives or offer to create

  1003→
  1004→```
  1005→> ch:^ +normalize

  1008→  Start with: dx:@file.csv
  1009→  Then use ^ to chain: ch:^ +normalize
  1010→```
  1011→
  1012→**Behavior:**
  1013→1. Check `.steno/current-session.json` for nodes
  1014→2. If empty, explain ^ and suggest starting fresh
  1015→3. If nodes exist but last has no outputs, use its inputs
  1016→

  1018→
  1019→```
  1020→> stat:ttest @baseline @new-data
